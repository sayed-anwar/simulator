Introduction to Multithreading in C++

Threads

A Thread is a part of our software (Code Segment) that can run independently of the rest of the process. For example, one process with two threads will share the Code Segment, Data Segment and Heap. Much of the Process State can be shared as well; however, each thread will need some thread-specific state information held aside. In addition, each thread needs its own Stack space to keep track of the function and method calls.

 

 

Because there is less work to be done to create a new thread (as compared to creating a new process), we can say typically using multiple threads will result in less overhead and more efficient processing. Going back to our image-processing example, our program might launch four threads to process different regions of the image. Since the image data is shared on the memory Heap, the threads do not need to pass anything between them using IPC.

Other common examples of multi-threaded applications include:

  • Database Management Systems
  • Web Servers
  • Web Browsers
  • Audio and Video encoding / processing systems

 

For purposes of our Financial Information Technologies course, we can highlight the way in which financial market data is consumed by client applications. A similar example is the use of multi-threading in algorithmic trading systems.
